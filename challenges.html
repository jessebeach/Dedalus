Challenges

Challenge 1 - Five Questions

In your group, invent a character. This character can be anyone, real or made up — a pirate, a misunderstood goblin, a wizard, a traveler, an athlete, a treasure hunter, a space explorer, or anyone else you imagine.

What are your character’s physical traits? Are they tall or short? Do they like to talk a lot or are they shy? What is this character wearing? Do they have any interesting possessions with them?

Now, think of 5 questions to ask this person and write them down.

Challenge 2 - Code your Character!

In this challenge, you will edit an HTML file. Using markup, you will define your character, the five questions and the answers to those questions.
  • Open the ‘Dedalus’ folder on USB drive.
First, try the example.
  • Open the file ‘character.html’ in the ‘examples’ folder in your editor.
  • Open the file ‘character.html’ in the ‘examples’ folder in Firefox. This will load the HTML page in the browser.
  • Find the menu trigger on the page with the label “person”. You can interact with this menu. Selecting items will load the character’s answer onto the page. Take some time to explore the example.
Then, build your own.
  • Open the file ‘index.html’ in the ‘my_game’ folder in your editor.
  • Open the file ‘index.html’ in the ‘my_game’ folder in Firefox. This will load the HTML page in the browser.
  • In your editor, find the HTML comment with the text “Characters”. This is where you will define your character in code.
  • Using the follow tags, you will code the questions you can ask your character and your characters answers to those questions. Use the code of the example character to guide you.
    • character[id][inventoryName]
      • <character id=“wizard” inventoryName=“Bellatrix”>{...actions}</character>
    • action[id]
      • <action id=“What is your name?”>{text}</action>
    • interact[with]
      • <interact with=“wizard”>{text}</interact>
  • Refresh the web page as your make changes to the ‘index.html’ file. The goal is to create a menu of questions that your character will answer.

Challenge 3 - Turn the Page

Add some details to the page you just wrote, where your character is introduced. What is the environment like around the character? Is the character in the middle of a task or activity? Write one or two paragraphs describing the setting.

Our game engine gives us the capability to write different pages and link them. Think of this like “turning a page”, but we can link any one page to any other page.

As a group, decide on what the next page will be. Does the character notice something and decide to investigate? Is the character on a mission and then move to the next stage? Does some event, like a shipwreck or an alien spaceship interrupt them and plunge them into the narrative?

Jot down a few notes about what might happen next.

Challenge 5 - Five actions

In your group, imagine on an object. Maybe this is an object that your character has. Or perhaps this is an object that your character will find.

What are the characteristics of the object? Is it big or small? Is it smooth or spiky? Is it safe or dangerous? Is it magical?

Then think about what your character can do with this object. One action may be to simply examine the object so that the character can learn about it. Maybe your character shake the object or try to weigh it in their hands.

One special action that our game engine allows us to do is put objects in an inventory. In this way, the object becomes available on all pages. Think about your inventory as a pocket or backpack. The instructions in Challenge 6 will explain how to perform a Take action.

Jot down some notes about your object.

Putting an object in the Inventory

Our game engine let’s us put objects in an inventory. The Inventory is available across all pages. Think of it like a pocket or bag of objects that your character has with them.
  • Open the file ‘object.html’ in the ‘examples’ folder in your editor.
  • Find the text characters ‘{{' and '}}‘. These characters are a special syntax in our game engine. The text side these characters must be code that the game engine can execute. We can use a special object named story inside these brackets. The story object has several methods that we can call. One of these methods is putInInventory.
    • story.putInInventory({id})
  • Add an object to the inventory and verify that it is there.
  • Create several more objects and add them all to the inventory.

Challenge 7 - Will they explode if we mix them together?

Our game engine let’s us define interactions between objects. Perhaps your character must find a key to open a box or a door. Maybe they need to build an ancient machine to open a doorway to another world. Or maybe they’re mixing chemicals together for a potion...but mix the wrong ones and poof!

In your groups, brainstorm how the objects in your game might interact. Are the interactions funny? Do they move the story plot along? Are you building a puzzle with lots of pieces that connect together?

Jot down your ideas.

Challenge 9 - So what happens next?

By this point you should have a character, a couple pages and an object. Now it’s time to think about what exciting adventures or mysterious discoveries will occur in your game!

A game is a logical set of options. We develop software in much the same way. We take account of the objects in our systems, the actions they have available and how these objects evolve over time with user interaction. 

In your group, brainstorm more pages, objects and characters. You can use the Monkey String and foam boards to create tokens to represent the pieces of your game. You can also map out the paths of your game. 

Challenge 10 - Code your game!

You have all the basic pieces you need to build a game — Characters, Pages and Objects! The real magic of a game is the imagination and story that you bring to it. Any good game engine just gives you to the tools to express your ideas. 

You can use the example games for more ideas and coding patterns.
